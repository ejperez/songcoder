What is Song Coder?

Song Coder is a chord sheet maker written in Javascript. It uses plain text for describing the music instead of a graphical user interface (GUI), which allows you to write charts faster. It allows you to focus on the actual content of the chord chart instead of how it looks.

Instructions

Let's assume that you want to chart the song "Happy Birthday". The source code of the song might look like this:

title:Happy Birthday
artists:Traditional
key:G
comment:Sample song
bpm:100
meter:4/4
body:
[Intro] G G7 | C Cm | G/D D | G

[Verse] G | D | D | G | G G7 | C Cm | G/D D | G

The source code is the plain text description of the song and its musical elements.

The following fields must be present (in exact order) in the source code:

title:
artists:
key:
comment:
bpm:
meter:
body:

All fields are required to have a value except the comment field. The value of a field must be entered on the right side of the colon (:), with the exception of the body field. Its content must be entered on a line below it.

Now let's talk about the syntax for the body.

[Intro] G G7 | C Cm | G/D D | G

[Verse] G | D | D | G | G G7 | C Cm | G/D D | G

Sections

A section of a song is declared using this syntax:

[SectionName]

If the section name contains several words, replace the spaces with underscore.

[Chorus_Higher]

Bars

The pipe symbol "|" is used to define the bars of a song. In the "Happy Birthday" song, the first line of the Intro section contains 4 bars, while the Verse section contains 8 bars.

Lines

If you want consistency with the count of bars per line (ex. 4 bars per line), we can split up the Verse section to multiple lines.

[Verse] G | D | D | G
G G7 | C Cm | G/D D | G

Chords

Chords must be separated by spaces.

G | D | D | G

Tranposing

On the lower-left part of the page, you can see the "Transpose to" option. You can select one of the keys listed there if you want to tranpose the chord chart to another key.

Printing the chord chart

Click the "Submit" to view your chord chart.

Repetitions

Back to the "Happy Birthday" example. Let's say you want to repeat the Intro section twice. You will notate it like this:

[Intro] [[: G G7 | C Cm | G/D D | G :]]

'[[:' tells what bar should the repetition start, while ']]:' tells to go back to the nearest bar with the start repeat sign ([[:) after playing the current bar.

If you want to repeat the bars more than two times, append the number to the end repeat sign (:]]) like this:

[Intro] [[: G G7 | C Cm | G/D D | G :]]3

Note Values

A note value is notated using numbers. See table below:

Numeric symbol Value
1 whole note
2 half note
4 quarter note
8 eighth note
16 sixteenth note

To add note values to a chord, append a colon (:) and a comma (,) separated list of note values. See example below:

[Intro] G:4,4,4 G7:4 | C Cm | G/D D | G

Rests

To indicate a rest, use the letter 'r'. Note values can also be added to a rest. See example below:

[Intro] r:1 | r:2 r:2 | r:4,4 r:4,4 | r:8,8,8,8 r:4,4

Dotted Timing

To add dotted timing to a note values, simply append a period (.). See example below:

[Intro] G:4.,4.,8 | r:4. G:4,4,8

Ties

To add a tie to a note value, append an underscore (_). See example below:

[Intro] G:4_,4, G:4_,4

To make a tie between two bars, add '[[_' to the bar to the right together with 'x' and the note values. See example below:

[Intro] G:4,4,4 G:4 | [[_ x:4 G:4,4,4

Comments

To add comments, type apotrophe (') followed by the comment. If the comment contains spaces, replace the spaces with underscore (_). See example below:

[Intro] 'Comment G C | G C

Use double qoutes (") to indicate a right-aligned comment.

[Intro] "Comment G C G C | G C

Other symbols

Symbol Description
[[ Double bar line - change in rhythm
[[(new key) Change in key e.g. [[Eb
